# Skill 19 — Beat Analysis in Dialogue

## Purpose

A *beat* is one exchange of action/reaction in character behavior. Every line of dialogue is an action taken toward a want — and every response is a reaction that advances, resists, or redirects. The *gerund* is the most important word: it names the ACTUAL action being taken, which is almost never the same as the literal speech.

If gerund and literal speech are identical — if a character is demanding by literally saying "I demand" — the dialogue has no inner life.

Run on all significant dialogue scenes.

## Inputs

- `script.txt`
- `15_scene_values.md`

## Instructions

### Step 1 — Select the scene
Each significant dialogue exchange. Prioritize scenes that failed the scene-value test — flatness there often has beat-level causes.

### Step 2 — Assign gerunds per beat
For each exchange (character speaks, other responds):
- Speaking character is DOING what (not saying what): demanding, flattering, deflecting, threatening, pleading, testing, stalling, exposing, retreating
- Responding character: ignoring, challenging, capitulating, redirecting, accepting, attacking, comforting, withdrawing

Write as pairs: demanding/ignoring, flattering/deflecting.

### Step 3 — Read the gerund sequence as a story
Stack pairs in order. Read as narrative. Shape? Escalation? Turning point?

### Step 4 — Find flat passages
Consecutive beats with essentially the same gerund pair = flat. Demanding/ignoring → demanding/ignoring → demanding/ignoring = looping, not progressing.

### Step 5 — Compare gerunds to literal speech
For each beat: how far is the gerund from the literal content? Large distance = strong subtext. No distance = on the nose.

### Step 6 — Locate the turning point
The beat where the scene's value shift occurs. Does the gerund sequence earn it?

## Output

Save as `19_beat_analysis.md`. For each dialogue scene:

- **BEAT MAP** — beat 1: [A gerund] / [B gerund]; beat 2: ...
- **TURNING POINT** — beat # + what shifts and why
- **FLAT PASSAGES** — beats that loop
- **SUBTEXT DISTANCE** — strong/weak; on-the-nose beats called out
- **TURNING POINT EARNED** — Y/N with explanation

Then overall:

- **STRONGEST SCENE BEAT MAP** — which scene, why
- **FLATTEST SCENE** — needs restructuring
- **PROBLEMS** — specific beats that are flat, on-the-nose, or unearned
- **SUGGESTIONS** — concrete (e.g., "Sc 5 beats 3-5 are all pleading/deflecting. By beat 3 this stops generating new info. Beat 4 needs Sebastian to do something unexpected — not deflect differently but a new action that forces Florence to change strategy.")
- **TEACHING MOMENT** — 4-part structure; gerund reveals what literal speech hides; dialogue as series of actions, not statements
