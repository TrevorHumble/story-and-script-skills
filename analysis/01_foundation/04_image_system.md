# Skill 04 — Image System

## Purpose

An image system is a recurring motif — visuals, objects, colors, sounds, textures, gestures — that accumulates meaning across the story beneath the level of plot. It operates subliminally. The audience absorbs it without being told. Sometimes the image system *is* the story: it carries meaning the plot alone cannot.

McKee's test: remove the image system; if the story loses something real, it's working. If nothing is lost, the recurrence is decoration, not a system.

## Inputs

- `script.txt`
- `02_controlling_idea.md`
- `03_story_spine.md`

## Instructions

### Step 1 — Inventory recurring elements
Don't filter. List everything that repeats: objects, colors, sounds, gestures, textures, animals, technology, settings.

### Step 2 — Group into clusters
Cluster recurring elements into categories. Look for patterns you didn't expect. Name each cluster.

### Step 3 — Test for accumulated meaning
For each cluster: does meaning deepen or shift each recurrence? Or does it just repeat? Accumulated meaning = system. Repetition without development = decoration.

### Step 4 — Identify the dominant system
Which cluster carries the most thematic or emotional weight? Name it. Describe what it's doing.

### Step 5 — Connect to controlling idea and spine
Does the dominant system connect to the controlling idea? Does it track the spine — shift in parallel with the protagonist's journey? If they pull different directions, the story may be telling two different stories at once.

### Step 6 — Removal test
Pick two or three key scenes. Mentally strip the dominant system. Does the scene lose resonance? If yes: it's working. If no: present but not load-bearing.

### Step 7 — Involuntary systems
Are there patterns that appeared without conscious intention pointing the story somewhere surprising? Often more true to the story's real meaning than the chosen ones. Note any.

## Output

Save as `04_image_system.md`:

- **RECURRING ELEMENTS** — inventory grouped
- **IMAGE SYSTEMS IDENTIFIED** — per system: elements, description, accumulating?
- **DOMINANT SYSTEM** — name + what it's doing
- **CONNECTION TO CONTROLLING IDEA** and **CONNECTION TO SPINE**
- **REMOVAL TEST RESULT**
- **INVOLUNTARY SYSTEMS** if found
- **PROBLEMS** — scene-specific (decorative repetition, contradictory systems, missed opportunities)
- **SUGGESTIONS** — concrete
- **TEACHING MOMENT** — 4-part structure; symbol (announced) vs. image system (subliminal); why involuntary systems are worth attention
