# Skill 24 — Postmortem

## Purpose

The postmortem synthesizes every output from the prior 23 exercises into one digestible document. Not a summary — a *prioritized action plan with teaching embedded*. Separates what is working from what isn't, orders problems by urgency, and gives the writer a clear picture of what the next draft needs.

This is the document brought to the rewrite. After this skill completes, the orchestrator builds the FigJam beat board from postmortem.md + directors_notes.md.

## Inputs

All 23 exercise outputs, all summaries (`summary_after_04.md`, `_13.md`, `_20.md`), plus:

- `script.txt`
- `beat_sheet.txt`
- `directors_notes.md` — **CRITICAL: canonical authorial state.** Many problems flagged in earlier exercises are RESOLVED here. Do NOT list resolved problems as open. When a finding has been addressed in directors_notes, treat it as a STRENGTH (the script's plan already accounts for it).

## Instructions

### Step 1 — Triage all problems
Read every output. List every problem. Sort into three tiers:

- **Tier 1 — FOUNDATIONAL** (fix first; affect everything else): structural and thematic problems. Weak controlling idea, broken spine, missing structural beats, flat antagonism, one-dimensional conflict. Without these, scene-level improvements won't hold.
- **Tier 2 — SCENE-LEVEL**: scenes that don't turn, rhythm problems, gap failures, text/subtext misalignment.
- **Tier 3 — DIALOGUE AND SURFACE**: on-the-nose lines, exposition delivery, beat flatness, trialogue opportunities. Polish last.

After tier sort, **find cross-skill contradictions** — places where two skills reached different conclusions about the same element. Name both, explain the disagreement, state which to trust and why. Do not silently pick one.

### Step 2 — Identify what is working
For each exercise: what is the script doing WELL, with scene/line references? What should be protected in the rewrite? Specifics only — "the gum's saturation in Sc 1, 3, 5 is a functioning image system that connects to the controlling idea" is sufficient. "The image system is strong" is not.

### Step 3 — Write the priority action list
For each Tier 1 problem: a specific rewrite instruction. Not "strengthen the crisis" — "The crisis in Sc 5 is currently Florence choosing to give Sebastian all her gum. This is not a genuine dilemma because the audience does not yet feel what the gum costs her. Before the crisis, establish one moment where the gum is the only thing that saves her — so that giving it away is a real sacrifice."

### Step 4 — Write the teaching section
For each major finding (positive or negative): brief teaching moment using the 4-part structure. Education, not criticism. Goal: writer understands not just WHAT to fix but WHY it matters.

### Step 5 — Rewrite roadmap
Brief roadmap: order of attack. Tier 1 → Tier 2 → Tier 3. For a short, this is a one-page plan.

### Step 6 — Closing note
One honest paragraph: not a grade, not a verdict — what this script is attempting and how close it is to achieving it. What is its strongest impulse? Where is it most itself? What does it need to become what it's trying to be?

## Output

Save as `postmortem.md`:

- **WHAT IS WORKING** — numbered, scene-specific, with directors_notes additions counted as strengths
- **TIER 1 — FOUNDATIONAL PROBLEMS** — only OPEN problems (not resolved in directors_notes). Per problem: what it is, why it matters, what to do.
- **TIER 2 — SCENE-LEVEL PROBLEMS** — same structure
- **TIER 3 — DIALOGUE AND SURFACE** — same structure
- **CROSS-SKILL CONTRADICTIONS** — required section. Each contradiction: skills involved, both conclusions, verdict + why.
- **TEACHING MOMENTS** — principles behind the major findings; 4-part structure each
- **REWRITE ROADMAP** — ordered steps; explicit scope (small adjustments / scene-level / structural rebuild)
- **A NOTE ON THIS SCRIPT** — honest, generous paragraph. Knowledgeable collaborator tone, not judge.

## Standards
- Every problem must have a specific rewrite instruction
- Every strength must be named specifically with scene references
- Teaching moments explain WHY not just WHAT
- The closing note is honest and generous — this is a developing writer
